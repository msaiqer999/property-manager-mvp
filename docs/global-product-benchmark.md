# Global Product Benchmark Framework

## Purpose

This document defines a structured benchmarking framework for Property Manager / المدير العقاري.

Benchmarking is used to understand global product patterns, user expectations, operational workflows, pricing signals, and technical architecture choices. It must not be used to copy competitors blindly or expand the product beyond its approved staged roadmap.

Every future benchmark entry must be based on verified official sources and must clearly separate observed facts from interpretation.

## Product Position

Property Manager / المدير العقاري is a property operations platform.

It is focused on:

- Individual landlords.
- Family property owners.
- Owner-managed teams.
- Rental operations.
- Contracts.
- Rent schedules.
- Payments and overdue tracking.
- Expenses.
- Reports.
- Documents.
- Activity logs.
- Future maintenance workflows.
- Future verified rental supply.
- Future aggregated and anonymized data intelligence.

It is not:

- A real estate sales platform.
- An open classifieds website.
- An open broker marketplace.
- A public listing platform where anyone can post any property.
- A tenant portal in Stage 1.
- A daily rental booking engine in Stage 1.

## Abu Dhabi and UAE Adaptation Criteria

When evaluating any global product pattern, judge whether it fits Abu Dhabi and UAE operating needs.

Use these criteria:

- Does it help individual landlords or family property owners in Stage 1?
- Does it support Arabic and English usage?
- Can it work well on mobile browsers?
- Does it respect workspace-level data isolation?
- Does it avoid exposing tenant, owner, contract, or financial data?
- Does it support owner-managed teams?
- Does it avoid open broker posting and uncontrolled classifieds behavior?
- Does it support long-term rental operations before future daily rental?
- Can it later support property management companies without redefining Stage 1 concepts?
- Does it require legal, regulatory, or payment handling decisions that need separate review?
- Can it be implemented simply enough for the Abu Dhabi MVP pilot?

This framework does not make legal claims. Legal, regulatory, payment, or compliance-sensitive features must be separately reviewed by qualified professionals before implementation.

## Quarterly Review Process

Benchmarking should be reviewed quarterly or before a major product phase.

Recommended process:

1. Select categories relevant to the next product phase.
2. Assign each platform to a reviewer.
3. Use official product pages, official documentation, official help centers, official pricing pages, or official release notes.
4. Record verification date and source link.
5. Describe only what was actually observed.
6. Map each observed feature to a real Property Manager user problem.
7. Assess Abu Dhabi and UAE relevance.
8. Assess architecture impact.
9. Classify the decision:
   - Adopt now.
   - Architect now, build later.
   - Monitor.
   - Reject.
10. Add accepted ideas to the feature decision framework before implementation.

## Decision Rules

- Do not add a feature just because a competitor has it.
- Do not add public listing behavior that conflicts with the verified rental market direction.
- Do not add sales modules.
- Do not add tenant portal behavior to Stage 1.
- Do not add daily rental workflows to long-term contracts.
- Do not add data intelligence that exposes private workspace data.
- Prefer simple operational improvements that help Abu Dhabi landlords first.
